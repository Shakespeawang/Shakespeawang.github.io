计算机网络——传输层

传输层协议

TCP UDP协议；

TCP：传输控制协议，需要将要传输的文件分段；客户端与服务器之间保持一个会话，直到数据可靠传输

例如：网页

netstat -n 用于查看建立的会话；

netstat -an

netstat -nb 查看建立会话的进程

telnet ip port 测试远程计算机某个端口是否打开

UDP：用户报文传输协议。一个数据包就能够完成数据通信，不需要建立会话，不需要流量控制，不可靠传输；

例如：DNS解析域名；QQ聊天；屏幕广播；

TCP/UDP协议加上一个端口，可以标识应用程序；

常见的应用层协议使用的端口：（传输层与应用层的协议）

http = TCP+80；https =TCP+443; 远程桌面连接RDP=TCP+3389；；ftp=TCP+21

共享文件夹=TCP+445； 发邮件SMTP=TCP+25； 收邮件POP3=TCP+110；telnet=TCP+23；

服务与应用层协议之间的关系：

服务使用TCP或UDP的端口侦听客户端请求；

客户端使用IP地址定位服务器，使用目标端口，定位服务；

可以在服务器网卡上设置只开放必要的端口，实现服务器网络安全；

传输层：服务端的数据+协议+端口；

网络层：在传输层基础上加上IP地址

更改服务使用默认端口：迷惑入侵者，使系统更加安全；

网络层协议与传输层协议的区别：

网络层：主机之间的逻辑通信；

传输层：提供进程之间的逻辑通信；

传输层的功能：传输层为应用进程之间提供端到端的逻辑通信（网络层是为主机之间提供逻辑通信）

传输层要对收到的报文进行差错检测；同时提供面向连接（TCP）与无连接（UDP）的服务

传输层的端口: 协议号 TCP 6 ； UDP 17； IGMP 1

端口 0-65535

熟知端口：0-1023 ； 登记端口： 1024-49151 客户端端口：49152-65535

UDP的主要特点：①无连接， ②尽最大努力交付，即不保证可靠交付，同时不使用拥塞控制；③面向报文；④支持一对一，一对多，多对一和多对多的交互通信；⑤UDP首部开销小，只有8字节

注：UDP计算检验和时，用到了网络层数据包首部的数据

传输控制协议TCP概述

TCP是面向**连接**的传输层协议；每条TCP连接只能有两个端点，每一条TCP连接只能是点对点的；

TCP提供可靠交付的服务，TCP提供全双工通信，面向字节流

*面向字节流的过程如下图*

TCP的连接：TCP把连接作为最基本的抽象；

每一条TCP连接有两个端口,连接的端点不是主机，也不是主机的IP地址，不是应用进程，也不是传输层协议端口，TCP连接的端点叫做套接字（socket）

套接字(socket)=(IP地址：端口号) 每条TCP连接唯一地被通信两端的两个端点（即两个套接字）所确定；

一、TCP如何实现可靠传输；

确认与重传机制，可以在不可靠的传输网络上实现可靠的通信。这种可靠传输协议常称为自动重传请求ARQ（Automatic Repeate reQuest） 重传的请求是自动进行的，接收方不需要请求发送方重传某个出错的分组。

连续ARQ协议

在发送窗口内部的数据包在发送端暂时不能删除，一旦发送出后，收到确认信息，发送窗口滑动，数据包才能删除

还有另一种累积确认。

序号是每数据包中第一个字节在整个文件中的位置，确认号是数据包中最后一个字节在整个文件中的位置；

数据偏移：用于记录从哪个偏移量开始有数据；，有四位，每一位的1表示4字节，其本身最大15，因此，最大60字节

上述中URG是数据包在发送端缓存区中，直接优先发送; FIN是数据包在接收端缓存区，直接优先被读出

TCP三次握手：XP系统计算机访问WEB服务器；

第一次：xp——>web 告诉web最大支持数据包MSS为1460字节，发送同步标记SYN=1; seq =x

第二次：web——>xp 告诉xp最大支持数据包MSS为1460字节,并告诉窗口最大缓存64240字节（如果超过这个还没有确认就暂停） ，发送同步SYN =1 ，ACK=1，seq =y，ack = x+1

第三次：xp——>web 告诉web最大缓存是65535字节（如果超过这个还没有确认就暂停发送）发送ACK =1，seq=x+1， ack=y+1；

注意：需要第三次握手的原因：为了实现可靠传输，发送方和接收方始终需要同步( SYNchronize )序号。 需要注意的是， 序号并不是从 0 开始的， 而是由发送方随机选择的初始序列号 ( Initial Sequence Number, ISN )开始 。 由于 TCP 是一个双向通信协议， 通信双方都有能力发送信息， 并接收响应。 因此， 通信双方都需要随机产生一个初始的序列号， 并且把这个起始值告诉对方。

因此，只有两次的握手的话，至多只有连接发起方的起始序列化能被确认，另一方的序列号则得不到确认，但由于通信双方都需要随机产生一个初始的序列号，并且把这个起始值告诉对方，因此需要三次握手
