1、索引相关

1.1 索引分类：

主键索引、唯一索引、普通索引、全文索引、组合索引

mysql的innodb会为唯一非空主键建立索引，即主键索引

全文索引:（5.6之后才支持）很少用，比如表中存的是英文文章，需要提取其中包含了java关键字，就需要用全文索引，但是有其他技术可以替代，比如java的ES

组合索引：多个列组合成索引，需要用最左前缀匹配原则

回表：select语句根据索引找到主键值，再根据主键值找到所查记录的过程就叫回表 ，即查了两个索引B+树 （extra中不会出现using index） ，与索引覆盖是相反的

覆盖索引：select语句的数据列只用从索引中就能取得，不必读取数据行，就是查询列要被所建的索引覆盖（在extra中会出现 using index）

最左匹配：

索引下推：会在存储引擎这一层按照条件进行筛选再返回到server层，而不用下推的话，就会先按照第一个条件筛选返回，再筛选第二个条件，依次 ； 索引下推的话，用explain时，extra中会有using index condition

用where条件语句的话，含有or的情况下，比如where a=1 or b=2；假设还有其他字段c…，如果（a，b）是组合索引，则查询不会走索引，如果a是独立的索引，b也是独立的索引就会走索引

在where条件语句中，尽量不要用表达式，会使索引失效，比如where a+1 = 3；但是where a = 3-1就没影响

前缀索引：可以查看重复次数，select count (*), left（字段名，前几个）as pref from 表名 group by pref；

1.3 索引匹配方法

假如有一个user表，四个字段：id、age、name、job； 有两个索引：主键索引id 和组合索引（age，name，job）

全值匹配：和索引中的所有列进行匹配，如select * from user where age =10 and name = ‘july’;

匹配最左前缀：只匹配前面几列，如select * from user where age = 10； 比如创建了（a,b,c,d）的索引，只有条件语句中用了a、ab、abc、abcd才会触发索引，但如果是ac、ad、acd之类的也会触发，但是只用了a的索引

匹配列前缀：可以匹配某一列的值的开头部分，如select * from user where name like ‘j%’； 但是like ‘%j%’就不会用索引了，只会去全表扫描

匹配范围值：可以查找某一个范围的数据 如 select * from user where name > ‘mary’ ; 是可以用索引的，但是如果是name>123 就不会用到索引了，因为会把name隐式转换，name原本是varchar，会转为int

精准匹配某一列并范围匹配另外一列：可以查询第一列的全部和第二列的部分，如select * from user where name =’july’ and age>20; 如果在后面再加上and job=’clerk‘ ，这个job的条件其实会失效，组合索引中的

只访问索引的查询：查询的时候只需要访问索引，不需要访问数据航，本质上就是覆盖索引，如select age、name、job from user where age =10 and name = ‘july’ and job=’clerk‘ ;

1.4 其他相关概念

聚簇索引：mysql中的主键索引，索引值与记录值在一起，只能有一个聚簇索引

非聚簇索引：索引值与主键值存在一起，再根据主键值查询聚簇索引，也就是需要两次访问索引B+树

innodb不支持哈希索引，用的是自适应hash；memory引擎支持hash索引

B+树索引的存储形式：有两个头指针，一个指向根节点，一个指向关键字最小的叶子节点
