HTTP协议

October 28, 2020 2131

webservice = http协议+xml

rest = http协议+json

1、请求信息与响应信息

请求

1）请求行

①请求方法 ②请求路径 ③所用的协议

请求方法：GET、POST、PUT、DELETE、TRACE、OPTIONS、HEAD

HEAD： 跟get方法一致，只是不返回响应体，只有响应行和响应头)

TRACE： 用来代理上网， 看代理有没有修改http请求，但是服务器未必开启该方法支持

OPTIONS：返回服务器有哪些方法是可以用的

PUT：put方法跟post方法一样，对数据进行修改，post侧重于数据的增加，put侧重对数据的修改

2）请求头信息

可以有Accept-language；Accept； Accept-Encoding；Connection；host

3）空行

4）请求主体信息（可以没有）

头部信息结束后与主体信息之间要空一行

GET方法请求

post方法请求

post方法请求中需要在请求体的头部中加上，内容类型，内容长度， 空格后再加上请求体

响应

1）响应行

①协议

②状态码：用来反映服务器的响应情况的

200 OK ;404 NOT FOUND

301/2： 永久/暂时重定向；也就是会转到另一个服务器中； （如果是post方法提交一个数据，用了重定向之后，在新的请求中，提交的数据会丢失）

304：未修改（比如之前已经请求，已经在客户端缓存中，客户端会保存一个上次修改的时间戳和记录时间，然后再次请求时，会对比这个记录的时间戳，如果没变的话就是未修改）

503：服务器暂时不可用

500：服务器内部出错

③状态文字

上面的ok ，not found都是状态文字

如HTPP/1.1 200 OK

2）响应头：key-value类型

常见的有，编码格式（deflate、gzip和zlib、sdch） 内容格式（application/x-www-form-urlencoded；multipart/formdata、text/html、application/json）

content-encoding: gzip

content-length: 1339

content-type: application/javascript

如Host：localhost

Content-type：text/html

Content-length:5

4）响应主体（可能没有）

4、http协议缓存

思考：如果有N台缓存服务器，那么这N台缓存服务器怎样处理主服务器上的文件？

1、是否需要缓存

2、需要缓存多久

打开缓存，即打开apache中的缓存模块，——>在.htaccess中设置ExpiresActive On

在响应头会有如下信息

cache-control为最大周期

expires：什么时候到期

如果是在集群中，缓存服务器得到此图片，将会认为生命周期内有效，在这生命周期内，如果有对其有请求，将直接从缓存服务器中获取，这样可以降低主服务器的压力。

5、http协议与内容压缩

保存网页源码为txt文件，就会发现其占用空间比content-length小很多，这是由于有content-encoding，在进行压缩，即为了网页在网络上的传输速度，服务器会对主体信息进行压缩；

常见的有gzip压缩，zlib压缩，deflate压缩，compress压缩，以及google chrome的sdch压缩
