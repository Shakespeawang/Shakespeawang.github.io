版本控制：版本迭代器

作用：多人开发管理工具，实现版本控制、

  • 本地版本控制、适合个人使用，

  • 集中版本控制、多人共用一个服务器、同步更新或上传自己的修改，SVN就是这种

有单点故障问题

  • 分布式版本控制

所有人本地都有历史版本，可以离线本地提交，只需在联网时push到响应的服务器或者其他用户那里即可；

所以只要有一个用户的设备没问题就可以恢复所有数据，但是会增加本地存储空间

git是目前世界上最先进的分布式版本控制系统

在任意目录下进入git bash，然后跟linux中很相近

相关理论

git本地有三个工作区域：工作区（working directory）、暂存区（stage）、本地仓库（repository），还有一个是服务器上的远程git仓库（directory）

工作区：平时放代码的地方

暂存区：实际是一个文件，临时存放改动，实际就是一个文件，保存即将提交的文件列表信息

本地仓库：安全存放数据的位置，这里有提交到所有版本的数据，其中head执行最新放入仓库的版本
